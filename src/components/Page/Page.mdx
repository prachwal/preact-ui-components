# Page Component

The Page component demonstrates a complete page layout combining the Header component with content sections. It showcases component composition and interaction testing patterns.

## Overview

The Page component represents a typical application page that includes navigation (Header) and main content. It demonstrates how to build complex UIs by composing smaller components and how to test user interactions.

## Interaction Testing

The Page component includes examples of Storybook's interaction testing capabilities. The "Logged In" story demonstrates how to test user interactions:

```javascript
export const LoggedIn = {
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);
    const loginButton = await canvas.getByRole('button', {
      name: /Log in/i,
    });
    await userEvent.click(loginButton);
  },
};
```

This test:

1. Finds the login button using accessibility role and text
2. Simulates a user click
3. Verifies the interaction works as expected
